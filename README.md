# secret_chat

Мобильное приложение на **Expo (React Native)** с модульной архитектурой, разделением логики на сервисы, API-клиенты, контексты, хуки и интерфейсы. Проект предназначен для разработки защищённого чата / мессенджера с современным UI и гибкой инфраструктурой.

---

## ✨ Основные возможности

- Expo + React Native приложение

- Структурированная архитектура: `api/`, `services/`, `context/`, `hooks/`, `utils/`

- Авторизация и работа с backend-API

- Работа с глобальным состоянием через контексты

- Поддержка сборки через **EAS Build**

- Типизация с помощью **TypeScript**

---

## 📁 Структура проекта

```
secret_chat/
│
├── app/                 # Экранные компоненты и маршруты (Expo Router)
│   ├── (auth)/          # Экраны авторизации
│   ├── (tabs)/          # Основные вкладки приложения
│   └── _layout.tsx      # Глобальный layout маршрутизации
│
├── api/                 # HTTP-клиенты, методы запросов к серверу
│
├── services/            # Бизнес-логика и работа с данными
│
├── context/             # Глобальные контексты (авторизация, настройки и др.)
│
├── hooks/               # Повторно используемые кастомные хуки
│
├── utils/               # Утилиты, вспомогательные функции
│
├── interfaces/          # TypeScript интерфейсы и типы
│
├── assets/              # Изображения, иконки и другие ассеты
│
├── app.config.js        # Expo-конфигурация
├── app.json             # Дополнительная конфигурация проекта
├── eas.json             # Настройки EAS Build
├── package.json
└── eslint.config.js     # Линтинг и правила стиля кода
```

## 🚀 Установка и запуск

### 1. Установите зависимости

```bash
npm install
```

### 2. Запуск в режиме разработки

```bash
npm start
```

Expo DevTools откроются автоматически.

### 3. Запуск на Android

```bash
npm run android
```

### 4. Запуск на iOS (macOS)

```bash
npm run ios
```

---

## 📦 Сборка через EAS

Перед сборкой настройте переменные окружения и конфигурацию в `eas.json`.

Запуск сборки:

```bash
eas build --platform android
eas build --platform ios
```

Авторизация:

```bash
eas login
```

---

## 🔧 Переменные окружения

Создайте файл `.env` в корне проекта (пример):

```
API_URL=https://your-backend.example/api
GOOGLE_CLIENT_ID=xxx.apps.googleusercontent.com
```

И добавьте `.env` в `.gitignore`.

## 🧱 Архитектура

Проект разделён на независимые модули:

### `api/`

Содержит чистые функции работы с backend:

- авторизация

- запросы к серверу

- обработка ответов


### `services/`

Инкапсулирует бизнес-логику, объединяет работу API и состояния.

### `context/`

Глобальное состояние:

- авторизация пользователя

- данные приложения

- настройки


### `hooks/`

Переиспользуемые логические блоки, например:

- работа с формами

- загрузка данных

- события UI
